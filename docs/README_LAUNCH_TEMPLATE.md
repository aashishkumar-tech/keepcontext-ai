# README Launch Template

Use this template when preparing a project README for launch.

```markdown
# <Project Name>

> <One-line value proposition>

## Official Links

- Repository: https://github.com/<org-or-user>/<repo>
- Issues: https://github.com/<org-or-user>/<repo>/issues
- Releases: https://github.com/<org-or-user>/<repo>/releases
- Docs: https://github.com/<org-or-user>/<repo>/tree/main/docs

## 60-Second Quick Start

```bash
git clone https://github.com/<org-or-user>/<repo>.git
cd <repo>
# setup commands
# run commands
```

Health check: <http://localhost>:<port>/health

## Who This Is For

- <target user 1>
- <target user 2>
- <target user 3>

## First Run Validation

1. Health endpoint returns 200.
2. First core workflow succeeds.
3. Output is visible in UI or API response.

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| <symptom> | <cause> | <fix> |

## Contributing

See CONTRIBUTING.md.

## License

MIT

```

## Before You Publish

1. Verify all links are public and correct.
2. Verify all commands run on a clean environment.
3. Verify default ports and URLs are consistent.
4. Verify root LICENSE exists and matches README claim.
